How it works

From prototype to trusted production

Four steps between a working demo and an agent you can actually ship with confidence.

01 — INSTRUMENT
Wrap any agent in one line
CortexTracer auto-detects LangGraph, CrewAI, or any Python callable. No refactoring, no decorators, no config files.
tracer = CortexTracer(project="my-agent") agent = tracer.wrap(your_agent)
02 — DEFINE
Write golden datasets in YAML
Define what correct looks like — expected keywords, tool calls, latency budgets. One YAML file per agent workflow.
cases: - id: refund_approved expected_output_contains: [refund, approved] expected_tool_calls: [lookup_refund]
03 — GATE
Block PRs when quality drops
One fail_on expression in GitHub Actions. When task completion drops below your threshold, the merge is blocked — automatically.
EvalSuite.run( dataset="golden_v1.yaml", fail_on="task_completion < 0.90" )
04 — OBSERVE
Live traces + Slack alerts
Every production run is traced — latency, tool calls, failure kind. Slack alert fires the moment something regresses.
GET /v1/traces?project=payments-agent → node waterfall, tool calls, failure detail

from cortexops import CortexTracer, EvalSuite

def my_agent(input_data):
    return {"output": "refund approved for your order"}

    dataset={
        "project": "my-agent",
        "cases": [{
            "id": "refund_test",
            "input": {"query": "process my refund"},
            "expected_output_contains": ["refund", "approved"],
        }]
    },
    agent=my_agent,
    verbose=True,
)
